# 📊 实时进度监控系统

## 功能概述

本系统提供实时的视频生成进度监控，让您能够清楚地了解《被讨厌的勇气》视频制作的每个步骤。

## 🎯 主要功能

- **实时进度显示**: 动态更新生成状态
- **进度条可视化**: 直观的进度条和百分比
- **时间预估**: 智能计算预计剩余时间
- **速度统计**: 显示处理速度和效率
- **多任务监控**: 支持音频、图片、视频渲染等多任务

## 🚀 使用方法

### 方法1: 分终端监控（推荐）

1. **终端A - 启动监控器**:
   ```bash
   npm run monitor-progress
   ```

2. **终端B - 开始视频生成**:
   ```bash
   npm run create-book-video "被讨厌的勇气"
   ```

### 方法2: 单终端查看

运行视频生成后，进度会自动写入到控制台和文件。

### 界面元素解释

- **已运行时间**: 从开始生成到现在的总时间
- **当前任务**: 正在执行的具体任务（脚本生成/音频生成/图片生成/视频渲染）
- **进度条**: 可视化进度条，█ 表示已完成，░ 表示剩余
- **进度统计**: 当前完成数量/总数
- **预计剩余**: 基于当前速度计算的剩余时间
- **处理速度**: 每秒处理的帧数或文件数

## ⚠️ 状态指示

- **🟢 正常**: 进度正常更新，显示绿色状态

## 🔧 故障排除

### 监控器不显示进度
1. 确保 `output/video_progress.txt` 文件存在
2. 检查是否有其他进程在运行
3. 重启监控器

### 进度更新停止
1. 检查视频生成进程是否还在运行
2. 查看终端是否有错误信息
3. 可能需要重新启动生成过程

### 时间预估不准确
- 初期预估可能不准确
- 随着处理的进行，预估会越来越精确

## 🎯 技术实现

- **实时监控**: 使用 Node.js `fs.watchFile` 监控进度文件
- **状态解析**: 智能解析进度信息，提取关键数据
- **UI渲染**: 控制台动态刷新，彩色显示
- **性能优化**: 高效的文件监控和状态更新

## 📈 性能指标

- **音频生成**: ~1-2秒/文件
- **图片生成**: ~3-5秒/张
- **视频渲染**: ~0.1-0.2秒/帧 (取决于复杂度)
- **整体流程**: 约10-15分钟完成整个视频

## 🎉 使用建议

1. **推荐使用双终端**: 一个监控，一个生成，便于同时查看
2. **监控器常驻**: 可以一直运行，随时查看进度
3. **定期检查**: 如果进度长时间无更新，请检查进程状态
4. **合理预期**: 视频生成需要时间，请耐心等待
